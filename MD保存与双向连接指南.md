# MD 文件保存与双向连接指南

## 1. 文件保存规范

1. 文件统一使用 `.md` 后缀。
2. 文件名尽量语义化，避免过长，建议：`主题-子主题.md`。
3. 推荐 UTF-8 编码，避免乱码。
4. 一篇笔记只聚焦一个主题，便于被其他笔记引用。
5. 修改后及时保存，避免只停留在编辑器缓冲区。

建议目录结构示例：

```text
考研/
  408/
    4.8数据结构.md
    4.9数据结构.md
  数学1/
    4.8线代复习.md
    4.9线性代数.md
```

## 2. 基础写作模板

```markdown
# 标题

## 核心结论

- 结论 1
- 结论 2

## 关键概念

- 概念 A：...
- 概念 B：...

## 关联笔记

- [[另一篇笔记名]]
```

## 3. 双向连接 [[]] 的使用

`[[...]]` 是知识网络里的“内部链接”。

1. 在 A 笔记写 `[[B]]`，表示 A 指向 B。
2. 如果系统支持反向链接（Backlinks），在 B 页面会自动看到“被 A 引用”。
3. 这就形成了双向关系：
   A -> B（显式）
   B <- A（反向自动发现）

### 3.1 基础语法

```markdown
[[线性代数]]
[[4.8数据结构]]
```

### 3.2 推荐命名策略

1. `[[标题即文件名]]`：最稳定。
2. 不要频繁改文件名；改名后请同步更新链接。
3. 若存在同名笔记，建议加前缀区分，如：
   - `[[数学-线性代数]]`
   - `[[408-数据结构]]`

### 3.3 连接示例

在 `4.9数据结构.md` 中：

```markdown
本节和 [[4.8数据结构]] 连续。
图的存储可对照 [[线性代数基础]] 的矩阵表示理解。
```

在 `4.8线代复习.md` 中：

```markdown
矩阵与图邻接矩阵可参考 [[4.9数据结构]]。
```

这样你的笔记会从“列表”进化为“网络”。

## 4. 常见错误与修复

1. 链接写成了文件路径：
   - 不推荐：`[[考研/408/4.8数据结构.md]]`
   - 推荐：`[[4.8数据结构]]`
2. 链接文本与文件名不一致，导致无法解析。
3. 文件名包含多余空格或特殊符号，导致匹配失败。
4. 创建了新笔记但未保存，系统无法建立索引。

## 5. 保存前检查清单

1. 当前文件是否已保存（Ctrl+S）。
2. 新增的 `[[...]]` 是否能跳转。
3. 是否出现孤立笔记（没有任何入链/出链）。
4. 是否至少关联 1-3 篇相关笔记。

## 6. 实战建议（针对考研知识库）

1. 每天新增笔记后，至少补两条内部链接：
   - 一条连“前置知识”
   - 一条连“后续知识”
2. 每周做一次“孤立笔记清理”，给无连接笔记补链。
3. 对高频主题建立索引页，如：
   - `[[知识目录]]`
   - `[[高数必背清单]]`

通过持续使用 `[[双向连接]]`，你的知识会从“文档堆叠”变成“可检索、可回溯、可扩展”的学习图谱。
