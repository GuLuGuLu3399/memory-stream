## Technical Details

### Why shallowRef?
For large graphs with 1000+ nodes, `ref()` creates deep reactivity on every element, causing:
- Memory overhead from proxy wrappers on each node/edge object
- Performance degradation during array mutations
- Unnecessary reactivity tracking when we always replace entire arrays

`shallowRef()` only tracks the `.value` reference itself, making it ideal for:
- Large arrays where we replace the entire array on updates
- Vue Flow node/edge arrays that get replaced during layout operations
- WebSocket incremental updates that use spread operator or filter

### Mutation Pattern Requirements
With `shallowRef`, reactivity only triggers when:
- `nodes.value = newArray` (whole array replacement)
- `edges.value = newArray` (whole array replacement)

NOT when:
- `nodes.value.push(newNode)` ❌
- `nodes.value[i].property = value` ❌
- `nodes.value.splice(...)` ❌

All code already followed the correct pattern, making this a safe optimization.
